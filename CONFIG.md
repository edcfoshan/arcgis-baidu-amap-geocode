# 密钥配置指南

本工具箱需要配置百度地图和/或高德地图的 API Key 才能使用。支持配置**多个 Key**，工具会自动轮换，提高并发处理速度并避免单个 Key 配额不足。

## 配置步骤

### 1. 复制模板文件

在 `config/` 目录下，有两个模板文件：

- `baidu_keys.example.txt` → 复制为 `baidu_keys.txt`
- `amap_keys.example.txt` → 复制为 `amap_keys.txt`

### 2. 申请 API Key

#### 百度地图 Key

1. 打开 [百度地图开放平台](https://lbsyun.baidu.com/apiconsole/key)
2. 登录百度账号，进入 **控制台** → **应用管理** → **我的应用** → **创建应用**
3. 应用类型选择 **服务端**
4. 启用以下服务：
   - 地理编码 V3
   - 地点检索 V3
   - 行政区划查询
   - 逆地理编码 V3
   - 路线规划 V1
   - 实时路况 V1
5. 将获取到的 AK 填入 `baidu_keys.txt`

#### 高德地图 Key

1. 打开 [高德开放平台](https://console.amap.com/dev/key/app)
2. 登录高德账号，进入 **应用管理** → **我的应用** → **创建新应用**
3. 服务平台选择 **Web 服务**
4. 启用以下服务：
   - 地理编码
   - 逆地理编码
   - POI 搜索
   - 行政区划查询
   - 路径规划
   - 输入提示
5. 将获取到的 Key 填入 `amap_keys.txt`

### 3. 填写 Key

将 Key 按行填入配置文件，每行一个。以 `#` 开头的行为注释，会被忽略。

**示例 `baidu_keys.txt`：**
```txt
# 百度 API Key - 个人开发者
AbCdEf1234567890GhIjKlMnOpQrStUv
# 百度 API Key - 企业账号
ZyXwVu0987654321TsRqPoNmLkJiHgFe
```

**示例 `amap_keys.txt`：**
```txt
# 高德 API Key - 主账号
a1b2c3d4e5f67890a1b2c3d4e5f67890
# 高德 API Key - 备用账号
f6e5d4c3b2a10987f6e5d4c3b2a10987
```

### 4. 工具箱中如何使用

按平台选择：

- **只用高德**：只需配置 `amap_keys.txt`
- **只用百度**：只需配置 `baidu_keys.txt`
- **都要/双平台**：两个文件都需要配置

### 5. 多 Key 的优势

- **提高并发量**：工具使用多线程并发请求，更多 Key = 更多可用 QPS
- **避免配额不足**：单个 Key 有日调用量限制（企业 30 万/天、个人 5000/天），多 Key 可分散调用量
- **自动容错**：当某个 Key 触发限流或配额耗尽时，自动切换到下一个 Key 并延迟重试

## 常见问题

### 提示 "Key 文件不存在"

检查 `config/` 目录下是否存在 `baidu_keys.txt` 或 `amap_keys.txt`。如果文件不存在，按照模板文件创建即可。

### 提示 "Key 文件为空"

检查文件中是否填写了有效的 Key，且 Key 没有被 `#` 注释掉。

### 高德返回 "USER_DAILY_QUERY_OVER_LIMIT"

高德单个 Key 的日调用量已用完。解决方案：
- 增加更多 Key
- 降低处理速度档位
- 等待次日配额刷新

### 百度返回配额相关错误

百度单个 Key 的日配额（PV）已用完。解决方案同上。

### 为什么需要多个 Key？

高德个人开发者日配额约 5000 次，企业 30 万次。百度的并发限制（QPS）较低。以地理编码为例，处理 1000 条地址需要 1000 次调用，单个个人 Key 很快耗尽。配置多个 Key 可以有效提高处理能力。
